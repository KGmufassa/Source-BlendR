## Quality Checklist

- [ ] SOLID principles applied where appropriate
- [ ] Functions are small enough to reason about
- [ ] Naming is explicit
- [ ] No magic numbers
- [ ] Error handling is structured
- [ ] Duplication is controlled
- [ ] Code is readable before clever
- [ ] Testability is preserved through dependency boundaries
