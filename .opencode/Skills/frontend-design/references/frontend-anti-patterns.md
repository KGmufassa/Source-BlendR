# Frontend Anti-Patterns

Use this file to diagnose weak design output and avoid generic generated UI.

## Common Failures

### Generic Hero + Cards Layout

Problem:
- interchangeable output
- weak product identity

### Decorative Gradient Overuse

Problem:
- visual filler without structure

### Default Font Dependence

Problem:
- no typographic intent

### Equal Visual Weight Everywhere

Problem:
- poor hierarchy
- unclear action priority

### Excessive Effects

Problem:
- polish without purpose
- lower usability on dense interfaces

### Inconsistent Spacing

Problem:
- weak rhythm
- amateur visual system feel

## Correction Strategy

- choose a stronger surface-specific composition
- define tokens before styling heavily
- reduce decorative clutter
- strengthen hierarchy through spacing, scale, and contrast
- make one or two distinctive decisions, not random ones

## Failure Repair Rules

- if a dashboard feels flat, increase zoning clarity before adding effects
- if a landing page feels generic, strengthen typography and pacing before changing colors
- if a settings UI feels confusing, regroup by user intent and move frequent actions back into context
- if dense data feels overwhelming, restructure filters and toolbar actions before reducing content
