# Surface Types

Use this file to choose the right design posture before committing to layout or aesthetic direction.

## Landing Page

Priorities:
- narrative sequencing
- emotional first impression
- conversion hierarchy
- section rhythm

## Dashboard

Priorities:
- fast orientation
- strong hierarchy
- action discoverability
- repeatable data zones

## Workspace / Tool UI

Priorities:
- shell clarity
- panel relationships
- visible system state
- focus continuity

## Settings / Configuration UI

Priorities:
- clear categorization
- progressive disclosure
- low-friction editing
- predictable structure

## Consumer App Surface

Priorities:
- immediate comprehension
- strong primary CTA
- low interaction friction

## Admin / Internal Surface

Priorities:
- control
- auditability
- dense status visibility
- efficient repetitive action handling

## Marketing + Product Hybrid

Use a split posture:
- marketing surfaces can be more expressive
- app surfaces should be more disciplined

Do not copy the same composition logic across all surface types.
