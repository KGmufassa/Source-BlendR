# Visual Directions

Use this file to choose a strong aesthetic direction that still fits the product surface.

## Selection Rule

Choose a direction based on trust level, energy, and task style, not trend alone.

## Clean Utilitarian SaaS

Best for:
- dashboards
- admin tools
- settings-heavy products

Strengths:
- clarity
- trust
- maintainability

Risks:
- can become bland if hierarchy and typography are weak

## Editorial

Best for:
- content-forward products
- knowledge tools
- narrative marketing surfaces

Strengths:
- strong typography
- pacing
- expressive hierarchy

Risks:
- can reduce operational clarity if applied too heavily to tools

## Premium Minimal

Best for:
- high-trust products
- polished consumer experiences
- selective product marketing

Strengths:
- focus
- elegance
- restraint

Risks:
- can under-communicate dense information

## Technical Console

Best for:
- ops tools
- monitoring
- infrastructure surfaces

Strengths:
- status visibility
- structure
- expert confidence

Risks:
- can become noisy or hostile if contrast and density are mishandled

## Creative Workspace

Best for:
- editors
- creation tools
- panel-driven experiences

Strengths:
- shell coherence
- clear tool and panel relationships
- strong product identity

Risks:
- can overcomplicate simpler products

## Output Expectation

When selecting a direction, specify:
- direction name
- why it fits the app type
- token implications
- composition implications
- major risk to avoid
