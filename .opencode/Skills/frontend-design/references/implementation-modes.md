# Implementation Modes

Use this file when adapting the design output to a specific frontend stack.

## React

- use functional components
- keep components modular
- separate structure from styling logic

## Next.js

- respect routing and SSR constraints
- keep page-level structure clear
- avoid client-heavy behavior unless needed

## Tailwind

- use utilities intentionally
- avoid unreadable class chains
- keep token and hierarchy discipline visible

## Vanilla HTML / CSS

- use CSS variables
- keep class naming structured
- preserve semantic HTML

## General Rule

Do not let the stack flatten the design quality into boilerplate UI.
