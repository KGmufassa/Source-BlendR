# Layout And Composition

Use this file for page structure, spatial hierarchy, density, and visual rhythm.

## Core Questions

- where does the primary action happen
- what is seen first
- what stays persistent
- what can be secondary or collapsible
- where should density increase or decrease

## Composition Modes

### Narrative Composition

Best for:
- landing pages
- storytelling surfaces
- editorial layouts

Traits:
- sequential rhythm
- section pacing
- expressive whitespace

### Operational Composition

Best for:
- dashboards
- admin views
- monitoring interfaces

Traits:
- clear zoning
- repeatable structures
- fast scanning
- supportive placement for filters, actions, and status

### Workspace Composition

Best for:
- side-panel tools
- editors
- multi-pane apps

Traits:
- shell-driven layout
- supporting panels
- persistent context
- visible relationships between primary and secondary panes

## Hierarchy Rules

- emphasize one dominant action path
- group related controls
- use contrast, scale, and spacing together
- avoid equal visual weight everywhere

## Density Rules

- dense interfaces need stronger grouping, not just tighter spacing
- sparse interfaces still need anchors and rhythm
- density should reflect task complexity

## Grid And Alignment Rules

- use column grids as the default for web applications
- use baseline rhythm to improve multi-column legibility
- align objects centrally when they vary in width, and align text left for readability
- use consistent horizontal rhythm to improve scanability

Source note:
- Fluent layout guidance emphasizes column grids, baseline rhythm, and alignment discipline for legibility.
